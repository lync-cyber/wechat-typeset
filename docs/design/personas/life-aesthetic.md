# Life Aesthetic · 视觉人格规范

> 本文档是 **设计规范**，不是实现。落地由工程 agent 按此规范生成 tokens / elementOverrides / containers / assets。
> 所有判断都在公众号硬约束下成立：禁 `font-family` / `position` / `float` / flex / grid / `@media` / `@keyframes` / `:hover` / `-webkit-*` / flex gap，布局只有 `table` / `inline-block` / `block` + padding + margin + border。SVG 禁 id / style / script / url()，stroke-width ≥ 1，`<text>` 不声 font-family，光栅后字号 ≥ 14，纯白用 `#fefefe`。

---

## 定位一句话

**生活美学不是"咖啡馆博主"，是"编辑部静物台"。** 参照坐标是 *Kinfolk*、*Cereal*、*Apartamento*、《暮しの手帖》、*Magnolia Journal*——是**冷米 + 灰青 + 器物速写 + 巨大留白**的出版物，不是米色毛玻璃 + 圆角 + 一束水彩鲜花的手账模板。

**拒绝坐标**：Canva "Slow Living" 模板、小红书 "ins 风"（奶油橙 + 哑光荧光）、文创店水彩花束、一切带"温柔 / 治愈 / 少女心"词汇的配色、Notion 周计划甜品色。以上词汇出现在任何设计讨论里，都算触发警报。

气质关键词：**干燥、克制、器物、布纹、窗光、不可爱**。注意"不可爱"不是冷酷——是像《暮しの手帖》那样端正得体、有家常但不撒娇。

## 1. Persona 三件套

### 1.1 Color story

当前配色（暖橙 `#d98141` + 深赭 `#b96234` + 芥黄 `#efb758` + 奶米 `#faf6f0`）整套就是**秋日咖啡馆菜单**。问题不是"颜色难看"，是**色温全在暖黄一侧**、**三色等饱和度**，一打开就知道这是 2019 年的 Pinterest "Cozy Vibes" 版。

生活美学的色彩纪律是**掺灰、降温、压饱和**：主色要像**上过釉但没上过蜡的陶土**，不能像**刚熬好的焦糖**。米底要像**亚麻布**或**未漂白图画纸**，不能像**奶油蛋糕**。

#### 新色值提案

| token | 旧值 | 新值 | 命名（内部沟通用） | 理由 |
|---|---|---|---|---|
| `bg` | `#faf6f0` | `#efece4` | 亚麻米 | 降黄、加一点冷灰。`#faf6f0` 在屏幕上几乎等同于 "牛奶白"，廉价感来自它太"甜"。新值偏近未漂白画纸，桌面摄影的背景布色 |
| `bgSoft` | `#f2ead8` | `#e6e2d6` | 旧棉布 | 与主底 8% 明度差，够分辨容器边界，不至于像"贴黄胶带" |
| `bgMuted` | `#eadfc7` | `#d8d3c2` | 窑灰 | 更深一档，用于 code 底与极少数深底容器 |
| `text` | `#3a2d20` | `#26221c` | 墨陶 | 更深更冷，脱离"巧克力色"联想。文字必须比所有装饰色都深 |
| `textMuted` | `#7a6a58` | `#706a5e` | 灰陶 | 压掉黄味，偏近"湿陶土" |
| `primary` | `#d98141`（暖橙） | `#9c6b4f` | 陶土褐 | **关键改动**：从"焦糖" → "未上釉的赤陶"。`#d98141` 是秋叶糖果色；`#9c6b4f` 是素陶杯、牛皮笺、麻线的色。拒绝甜味，拥抱质地 |
| `secondary` | `#b96234`（深赭） | `#6b7a72` | 青瓷灰绿 | **180° 反转方向**：从"更深的暖"换成"冷灰绿"。这是 Kinfolk / Cereal 色板的签名冷色——青瓷、苔藓、厨房窗边的绿植在哑光光线下的样子。与 primary 的陶土形成"陶 + 瓷"的器物对照 |
| `accent` | `#efb758`（芥黄） | `#a89775` | 亚麻线 | 降一大档饱和度。accent 不再承担"亮点"职责，改承担"中性高光"——像亚麻布上的纬线、米纸的折痕。真正的重音交给**质地 + 留白**，不靠高饱和度点缀 |
| `border` | `#e0d1ba` | `#c9c2b2` | 细棉线 | 降一点彩度。所有线条都要看起来像**铅笔线 / 针脚线**，不是"彩色油漆刷出来的边框" |
| `code` | `#b96234` | `#706a5e`（= textMuted） | 灰陶 | **拒绝让 code 承担暖色**。代码在生活语境里是"烹饪计量单位"——15g、180°C、3min——是信息不是装饰 |

#### "拒绝什么"

- **拒绝 `#ffb084` / `#f5a97e`**（奶油橙、哑光荧光橙）。这是小红书 "ins 风" 的签名色，一出现就定性低了。
- **拒绝 `#faecd8` 奶米 + `#c8a15a` 浅金 + `#d4a95c` 亮金** 的组合。这是文创店礼盒内衬配色。
- **拒绝莫兰迪雾粉 `#e5c6c0` / 雾紫 `#c9bccb`**。那是 2018 年"高级灰"口红营销带坏的一整代配色。
- **拒绝所有饱和度 > 60% 的绿色**（包括松绿、墨绿、橄榄）。secondary 必须是**灰绿**，不是绿。
- **拒绝纯白 `#fff` 作为任何底色或留白容器**。公众号本身白，`#fefefe` 也只在 qrcode 内衬偶尔出现。生活美学的"白"永远是**米白 / 亚麻白 / 陶瓷白**。

#### 语义色（tip / warning / info / danger）

必须满足：亚麻米 `#efece4` 上正文对比度 `≥ 4.5:1`（AA）。更重要的是——**让它们别变成"医院 UI" 的冷冰冰四件套**。

策略：**从厨房 / 衣橱 / 纸品的实物色里取色**，而不是从 Bootstrap 调色板里取色。

| 语义 | accent | soft | 对比度 | 命名 | 取色来源 |
|---|---|---|---|---|---|
| tip（小记） | `#6f8268` | `#dfe4d8` | 4.8:1 | 菜叶绿 | 西芹、苔藓、厨房窗台小盆栽的叶背面 |
| warning（留心） | `#9a7b3e` | `#e8dfc4` | 4.9:1 | 黄铜色 | 老黄铜把手、蜂蜡、陶瓷上的刷色釉 |
| info（案头） | `#5c7283` | `#d8dde3` | 4.7:1 | 石板蓝灰 | 石板屋顶、墨水瓶、晨雾里的窗 |
| danger（慎） | `#8c4a3f` | `#e5cfc9` | 5.1:1 | 赤陶深 | 未上釉的深陶、烤盘底、铁锈边缘 |

#### 四个语义色在"禁欲"调性里不变冷冰冰医疗感——三条纪律

1. **全部掺灰**：没有一个 accent 饱和度超过 35%。Bootstrap 的四色之所以像医疗 UI，是因为它们都是**信号灯色**（纯红 / 纯黄 / 纯绿 / 纯蓝）。我们改用"哑光实物色"——每一个都能对应到一件你家里真实存在的器物。
2. **soft 底都要"脏"**：不用 `#e8f5e9` 这种淡薄荷糖底，而是**在 tip.accent 里加 40% 米底混合出来的"被米底晕染过的绿"**。看起来像是茶渍、油渍、或者布料本身的淡色纬线，而非"这里是提示框"的塑料感。
3. **形状才是主角，颜色只是标签**：见 §2 四件套——tip / info / warning / danger 在形状上已经能分辨，颜色只是**二次确认**。这样四色都可以保持低饱和度，不需要靠高饱和抢眼。

### 1.2 Typographic voice

literary-humanism 走的是"古籍大字距 + 2.0 行高 + 书斋气"。life-aesthetic 不能再走这条路——再走就是同一件事做两遍。

**我们走"杂志页眉"的路**：超大字距 + 中性行高 + 标题与正文的大字号落差。参考对象是 *Cereal* 和 *Apartamento* 的内页：h1 是一个**稀疏、克制、像展签**的东西，正文是**紧凑、中性、像说明文**的东西，两者咬合才是"编辑部感"。

#### Scale

| 层级 | px | font-weight | letter-spacing | line-height | 颜色 | 说明 |
|---|---|---|---|---|---|---|
| h1（展签） | 28 | 500 | 4px | 1.35 | `text` | **字重压到 500 而非 700**。杂志封面大标题的高级做法是"稀 + 不过粗"。28px + 4px 字距是整个主题最有辨识度的一手 |
| h2（栏目名） | 18 | 600 | 3px | 1.5 | `text` | 比 h1 小一档、字重反而高一档。h2 是"本栏目"标签，它应该**紧、密、低**，和 h1 的"稀、疏、高"形成对位 |
| h3（条目） | 15 | 600 | 1.5px | 1.6 | `text` | **与正文同号**。只靠字重 + 字距区分。h3 前允许一枚 2×8 的 `primary` 色小方块（见 §1.3 hairline bracket） |
| p（正文） | 15 | 400 | 0.8px | 1.75 | `text` | **baseSize 从 15 保留、行高从 1.9 降到 1.75**。1.9 是书斋行高，1.75 是杂志行高。生活美学是杂志不是书斋 |
| small（注脚） | 12 | 400 | 1.5px | 1.6 | `textMuted` | 用于日期、材料表、photo credit 式说明。字号小、但字距要大——小字拉开字距才像"印刷注释"，不像"手机备注" |
| pull-quote（栏目引言） | 20 | 400 | 2px | 1.5 | `textMuted` | **金句不靠粗体、不靠主色**——靠字号 + 字距 + textMuted 色，造出"展签旁的说明牌"感。这是与 literary-humanism "镇纸 22/500" 金句的根本分野 |

#### 字重对比策略

- 只用 **400 / 500 / 600 三档**，**不用 700**。700 在生活美学里永远过重——它自带"广告"气息。
- h1 用 500、h2 用 600，是故意的**反直觉**：大的反而不粗，小的反而粗。这造出"杂志页眉"特有的节奏。
- 禁止任何 **"主色 + 粗体 + 大字号"** 三元组合——那是促销海报。

#### 首段与段落策略

- `intro` 首段走**全大字距**路线：字号 15、letter-spacing 2px、textMuted 色、段左右各 0 内缩（不像 literary-humanism 那样缩进）。看起来像**杂志开篇的 "lede" 段落**——稍远、稍淡、稍稀。
- 正文段落**不做首行缩进**。段间距用 18px，比 literary-humanism 的 22px 紧——杂志版面比书斋紧。
- `section-title` 与后续段落之间允许 **32px 大间距**，用"呼吸"做章节分隔，不靠分割线。

#### 与 literary-humanism 的对位表（防止两套风格混淆）

| 维度 | literary-humanism | life-aesthetic |
|---|---|---|
| 参照 | 三联书店、岩波文库 | Kinfolk、Cereal、暮しの手帖 |
| 行高 | 2.0（书斋） | 1.75（杂志） |
| h1 字重 | 700 | 500 |
| h1 字距 | 3px | 4px |
| 主色取向 | 油墨褐 | 陶土褐 |
| 冷色 | 青瓷釉 3d4a3d（绿） | 青瓷灰绿 6b7a72（绿灰） |
| 装饰 | 云头 / 钤印 / 回纹 | 器皿侧剖 / 窗格 / 薄切线 |
| 语汇 | 题解 / 批注 / 校勘 | 小记 / 留心 / 案头 / 慎 |

### 1.3 SVG motif

**这是升格最狠的一刀**。现有 11 件资产里，叶子 h2Prefix、缠枝 dividerFlower、花瓣 dividerDots、豆芽引号 quoteMark、叶片 sectionCorner——**五件植物 motif 砍到一件**。剩下的位置留给**器物几何**。

升级原则：**硬边、细线、几何化、器物意味**。别再画"自然物"，去画"被人做出来摆在桌面上的物"。

#### ① h2Prefix：hairline bracket（薄切线）—— 取代叶子

```
 ┌─
 │    栏目名
```

- viewBox `0 0 14 18`，左上一个 L 形铜线：水平线 x=0→10 y=2，竖线 x=2 y=2→16，stroke `primary` 1px
- **为什么**：Cereal 页眉的签名符号就是这种细铜线括弧。它代表"这里开始是一个被框起来的内容"，比叶子克制一百倍
- 彻底删除当前的"叶片 + 叶脉"。叶脉是印刷品最廉价的信号

#### ② dividerGlyph：器皿侧剖线（替代 dividerFlower 缠枝）

```
    ╭───────╮
    ╰───────╯
   ─────────────
```

- viewBox `0 0 240 22`
- 中央一个**茶杯 / 陶碗的侧剖线稿**：宽 36、高 14，仅外轮廓，stroke `primary` 1.2px，无填充
- 两侧 `border` 色 1px 长线各 92px
- **提供两个变体**：碗（扁圆）、杯（窄高）。随机或按模板指定
- **为什么**：器皿侧剖是《暮しの手帖》与 Apartamento 的核心语汇。它不是装饰，是"主题标识"——告诉读者"这是一本关于器物与日常的刊物"

#### ③ dividerRule：窗格十字线（替代 dividerDots 散花瓣）

```
    ─────────── ┼ ───────────
```

- viewBox `0 0 240 12`
- 中央一个 8×8 的十字：横线 x=116→124 y=6、竖线 x=120 y=2→10，stroke `primary` 0.9px
- 两侧 `border` 色 1px 长线各 108px
- **为什么**：窗格十字线是编辑版面的经典"对齐标记"——像摄影棚的中心十字、像窗棂的交点。比"散落花瓣"克制太多

#### ④ dividerWave：**保留唯一植物 motif**，但必须改造

这是我留给植物的**唯一一个位置**。为什么留：生活刊物完全无植物意象会太冷——需要一件作为"温度锚"。但必须改造：

- **当前问题**：手绘波浪 + 渐淡末端 + stroke 1.4 + opacity 0.7，每一项都是"手账感"的信号
- **改造**：改为**一根极其简化的带叶枝条**——一条 1px 直线横过版面，单侧三片**菱形叶**（不是椭圆叶）各 4×2，叶子仅有轮廓不填充，`secondary`（青瓷灰绿）色。**几何化的植物**，不是"手绘植物"
- 每篇最多出现一次，且只出现在文章主体中部、不上 footer

#### ⑤ quoteMark：编辑引号（替代豆芽双引号）

```
 〔            〕
```

- viewBox `0 0 16 20`，**开方括号的上半角**：两条直线组成的┏形，2px 高、6px 宽，stroke `primary` 1px
- 闭方括号对应的镜像在 quoteCard 右下角出现
- **为什么**：豆芽引号是手写感最强的装饰，它在 Canva 模板里被用烂了。改为**编辑引号 〔 〕**，这是日式杂志与中文老式出版物共通的书面引号

#### ⑥ sectionCorner：信封折角（替代叶片角花）

```
 ╲
  ╲___
```

- viewBox `0 0 18 18`，右上到左下一根 0.8px 折线，左下连到一个小三角（底边 6、高 4）
- `primary` 色描边，**无填充**
- **为什么**：信封折角是生活刊物里"一封信 / 一则短札"的视觉语汇。它暗示"这是一段手写体量的内容"，比叶片角花叙事性强得多

#### ⑦ stepBadge：**全部重做**——从软圆双环改为"日课格"

```
 ┌──┐
 │ 1│
 └──┘
```

- viewBox `0 0 20 20`，**方框**（非圆！）：`rect 0.5,0.5 w18 h18` stroke `primary` 1px，**无圆角**
- 数字 1 / 2 / 3 在方框内，font-size 13 font-weight 600 fill `primary`
- **不填充、不阴影、不双环**
- **为什么**：软圆双环是 "App onboarding" 审美。方格是**日课表 / 日历格 / 收纳盒**审美。当 steps 容器代表"一日三时"或"一周七事"时，方格比圆章恰当一百倍
- 可选变体：方框左上角加一道 2px × 2px 的切角（模拟日历页脚折痕），作为"当天"标记

#### ⑧ 新增 `listBullet`：小方点

- viewBox `0 0 6 6`，`rect x=1 y=1 w=4 h=4 fill=primary`，**实心方块不是圆点**
- 用于 `ul` 列表项的前缀，取代默认的 `·` 圆点
- **为什么**：方块 bullet 是编辑版面的签名做法，圆点 bullet 是 Word 默认值

#### 图标（tip / warning / info / danger）——**全部去圆**

当前四个图标都是"圆形外框 + 对勾/感叹号/i/叉"，典型 Material Design。改造为：

- **全部去圆圈外框**，只保留内部符号
- tip：一根 0.8px 短横线 + 一个 2×2 小方块（像"笔记本上的项目符号"）
- warning：一个等边小三角（8×7，仅轮廓）——三角形自带"注意"语义，不需要叹号
- info：一个竖线 `|` 2px 粗 10px 长——像书本侧边的 "marker tab"
- danger：一个 × 号（两根 1.2px 交叉线），**不加框**——最极简的"此处有问题"信号

尺寸统一 14×14，颜色跟随各自 status.accent。

## 2. 19 个 container 的视觉差异化方案

### intro · 短札

**角色重命名**：**短札**（原"引子"）。不是"开篇",是"翻开刊物第一页的编者札记"。

**结构**：
```
  ┌─  （薄切线）
  │   短札正文，textMuted 色，字距 2px，
  │   两侧内缩 0（不内缩），段上下
  │   各 14px 呼吸
  │                              ─┐
  │                               │
```

- 左上、右下各一枚 `sectionCorner`（信封折角），**对角线呼应**
- padding `20px 24px 20px 24px`
- margin `0 0 32px 0`
- **无底色、无边框**——只靠对角两个折角定义空间
- 文本 textMuted + letter-spacing 2px

### author · 署名

**角色重命名**：**署名**。

**结构**：单行 inline-block，左侧 2px × 14px `primary` 色小竖条 + 作者名（14px / 500 / text 色） + " · " + 角色（12px / 400 / textMuted / letter-spacing 1.5px）。
- padding `4px 0 4px 10px`（左 10px 给竖条留位）
- **无边框、无底色、无圆角**
- margin `0 0 20px 0`

### cover · 扉页

**角色重命名**：**扉页**。

**结构**：
- 图片 `border-radius: 0`（**零圆角**，这是生活美学的一条红线）
- 图下紧跟一行 12px / textMuted / letter-spacing 2.5px / text-align:center 的小字题记（像 photo credit）
- 题记前加一枚 `listBullet` 小方块作为"图注起点"标记
- margin `0 0 36px 0`

### section-title · 栏目名

**角色重命名**：**栏目名**。

**variant 选择**：`cornered`。

**结构**：
```
 ┌─
 │   栏目名  （18px / 600 / 3px 字距）
 │
 ────────────────────────────
```

- 左上 `sectionCorner`（信封折角）
- 下方 1px `border` 色细线 **通栏**
- margin `36px 0 16px 0`
- **不用大字号**——section-title 用 18px（和 h2 同）、靠**上下留白 + 折角 + 通栏线**凸显层级

### quote · 引文

**角色重命名**：**引述**。

**variant 选择**：`column-rule`。

**结构**：
- 左右各一根 1px `border` 色竖线（**不是 primary 色**——引述不是重点，别抢戏）
- padding `10px 28px`
- textMuted 色 + letter-spacing 1px
- 首尾**不加编辑引号 SVG**——引号留给 quoteCard

### quoteCard · 栏目引言

**角色重命名**：**栏目引言**（不是"金句卡"——生活刊物没有"金句"这种东西，只有"开栏小语"）。

**variant 选择**：`frame-brackets`。

**结构**：
```
 〔
 
   引言正文 20px / 400 / 2px 字距 / textMuted
   三到五行内，不超过 60 字
   
                               〕
                        ——出处 12px
```

- 四角**不用** L 形 bracket，改为**仅左上一枚 〔 + 右下一枚 〕**（即 §1.3 编辑引号对）
- padding `28px 30px 24px 30px`
- **无边框、无底色**——靠两枚引号和大量留白定义空间
- 出处小字 12px / textMuted / 右对齐 / 前缀 `——`
- margin `28px 0`

**与 literary-humanism quoteCard 的差别**：那边是**首字下沉 + 朱 + 边框 + bgSoft**（像镇纸压在纸上），这边是**无框 + 大留白 + 引号对 + textMuted**（像展签旁的说明牌）。

### footerCTA · 卷末

**角色重命名**：**卷末**。

**结构**：
- 顶部 `dividerGlyph`（器皿侧剖线）
- 中间 12px / textMuted / letter-spacing 2px / 居中 文字
- 底部一行**极小**的日期 / 期号式信息：`Issue 023 · Spring 2026`（由模板生成）——10px letter-spacing 3px
- padding `32px 0 24px 0`，**无底色、无边框**

### recommend · 本期书单 / 物单

**角色重命名**：**本期书单** 或 **本期物单**（取决于内容）。

**结构**：
- `bgSoft`（旧棉布色）底
- 标题行："本期推荐" 14px / 600 / 3px 字距 / primary 色
- 列表项每行前加一枚 `listBullet` 小方块（**非箭头 ▸**），行高 1.9
- padding `18px 22px`
- radius 0
- **无阴影**

### qrcode · 印章位

**角色重命名**：**印章位**（二维码在生活刊物里就是"期刊社印章"的位置）。

**结构**：
- 居中 block
- 图像外框 1px `border` 色，内衬 padding 6px `#fefefe`
- 图像下方一行 12px textMuted letter-spacing 2px 文字，居中
- **不要任何装饰 SVG 包裹**——QR code 本身已经是最强烈的视觉元素，再加装饰就乱了

---

### tip / info / warning / danger · 四件套（**生活刊物的四种"留言条"**）

**核心原则：靠形状差异一眼可辨**。四者的结构策略都不同，即便去色打印也能区分。

| 语义 | 角色名 | variant | 形状策略 | 边框 | 标题前缀 | 留白 |
|---|---|---|---|---|---|---|
| tip | **小记** | `minimal-underline` | **最轻**。无边框无底色，标题下一道 1px tip.accent 短线（标题宽度的 60%） | 无 | "— 小记" 短破折号前缀 | padding `6px 0 14px 0` |
| info | **案头** | `accent-bar` | 左侧 2px info.accent 竖条 + info.soft 浅底 | 无外框 | "案头　" 两字加粗 600 + 两空格 | padding `12px 18px 12px 20px` |
| warning | **留心** | `pill-tag` | **顶部一枚悬出的"便签条"**：一个 warning.accent 色的窄条（高 6px、宽 42px）悬在容器顶部上沿，margin-top -3px | 1px warning.accent 实线框 + warning.soft 浅底 | 便签条内写"留心"（white text 12px 600） | padding `16px 20px` |
| danger | **慎** | `ticket-notch` | **最重**。顶部一条 2px danger.accent 粗线 + 底部一条 2px 粗线 + 左右无边框 + danger.soft 浅底。像**两道警示胶带**贴在容器上下 | 上下 2px 实线（左右无线） | "慎" 单字 + 全角冒号 + 内容，"慎："用 danger.accent 600 | padding `18px 22px` |

**一眼可辨的锚点**（去色打印也成立）：
- tip = **一道下划线**（最轻，像铅笔划）
- info = **左竖条**（像书页侧边标签）
- warning = **顶部便签条**（最跳，像贴纸悬出）
- danger = **上下双粗线**（最重，像胶带封条）

**语言层的差异化**（四者前缀完全不同的字符形状）：
- 小记：破折号 —
- 案头：两字 + 空格
- 留心：便签内白字（整个前缀是一枚色块）
- 慎：单字 + 冒号

**为什么不是默认的"四个同形只换色"**：默认主题的做法是偷懒，在生活刊物这种讲究质地的语境里是降级。每个 admonition 应该像**书桌上四种不同的留言纸**——一张便笺、一张明信片、一张贴纸、一张封条——形状不一样，读者才会**多读一眼**。

---

### highlight · 重音段

**角色重命名**：**重音**。

**结构**：
- **无边框、无底色**
- 左右各 28px 内缩
- 字号 16（比正文 15 大 1px）、字重 500、letter-spacing 1.2px、color `text`
- margin `20px 0`
- **靠内缩 + 字号升 1px + 字距升 + 字重 500** 四件事完成重音，不用色块

**为什么**：生活刊物的"重音"不喊不闹，它**比周围安静地"抬头"一点**。这是与 literary-humanism 的 highlight 的分野——那边用 bgSoft 底，这边什么都不用。

---

### compare · 两种做法（不是 PK）

**角色重命名**：**两种做法** 或 **并置**（绝不是"对比 / 对决 / PK"）。

**variant 选择**：`column-card`（结构正确，但要重新填充语义）。

**结构**：
```
 ┌─           ─┐     ┌─           ─┐
   做法甲              做法乙
   
   材料、步骤、结果     材料、步骤、结果
   
 └─           ─┘     └─           ─┘
```

- 两栏 `display:table-cell`，中间 18px gutter（用 margin 不用 gap）
- **每栏外框**：只画四角的 hairline bracket（每角 8×8 的 L 形细线），中间无边框——这是生活刊物展台的"产品围栏"画法
- padding `18px 20px`
- 栏标题 14px / 600 / primary 色 / 3px 字距，如"做法甲" / "做法乙"
- **拒绝 `ledger` variant**——绿红账本是效率软件，不是生活美学
- **提供 `stacked-row` 作为 fallback**：窄屏下上下堆叠，每张卡片仍走四角 bracket 画法

### steps · 一日三时 / 按序三则

**角色重命名**：**一日三时** 或 **按序**（根据模板选），**不叫"步骤"**。

**variant 选择**：`timeline-dot`（升级版：用 `stepBadge` 方格替代圆点）。

**结构**：
```
 ┌──┐ 
 │ 1│  晨 · 标题
 └──┘ 
       正文内容，左侧与方格对齐 28px
       
 ┌──┐ 
 │ 2│  午 · 标题
 └──┘
 
 ┌──┐
 │ 3│  夕 · 标题
 └──┘
```

- 左列 28px 宽，放日课格 stepBadge
- **方格之间不画连接线**（与 literary-humanism 的竖线连接不同）——生活刊物的"三时"是独立的三张照片，不是流程图
- 每步标题前缀由模板决定（"晨 / 午 / 夕"、"一 / 二 / 三"、"春 / 夏 / 秋"），默认用汉字而非阿拉伯数字
- margin between steps: `18px 0`
- **拒绝 `ribbon-chain`**——横向色带链是课程卡片审美

---

### divider · 呼吸

**角色重命名**：**呼吸**。

**variant 推荐优先级**：`glyph`（器皿侧剖）> `rule`（窗格十字）> `wave`（几何化植物枝）> `dots`（小方点阵）> `flower`（已废弃）。

- `glyph` 默认：器皿侧剖是这个主题的签名符号，每篇出现 1–2 次
- `rule` 次选：最克制的分割，两段同类内容之间用
- `wave` 特殊场合：想要"季节感"时用，每篇最多一次
- `dots` 备用：连续短段落之间的轻呼吸
- `flower` **永不再用**——原缠枝已废弃

上下 margin 统一 `28px 0`（比 literary-humanism 的 36px 紧，杂志节奏）。

---

### mpvideo / mpvoice · 嵌入件

**结构**：上下 1px `border` 色通栏横线，margin `20px 0`，padding `8px 0`。**不加底色、不加圆角、不加阴影**。平台组件本身不干净，主题能做的就是给它一个**安静的画框**。

### free · 自由容器

**结构**：padding `14px 22px`，margin `18px 0`，**无边框无底色无装饰**。兜底容器就该是一张白纸。

## 3. 反例对照（如果做糟会长什么样）

我要把四种常见翻车钉在耻辱柱上——每一种都有对应的规避动作：

### 3.1 "Canva 咖啡馆博主" 翻车版

**画像**：奶油橙 `#f5a97e` + 米白 `#fff8ee` + 圆角 16px + 每个 h2 前一片水彩小叶子 + 正文用 14px 手账字体（公众号会 fallback 回系统字，但主题作者还是加了 font-family）+ footer 一只大大的手绘咖啡杯 + divider 全程手绘波浪。

**规避**：
- 主色改陶土褐 `#9c6b4f` 而非奶油橙
- 圆角全部归零（radius sm=0 / md=0 / lg=2，只在 author 内缩方格极窄场景用 2px）
- 装饰砍到器物几何，叶子/花瓣/波浪全部下课
- 禁止 font-family（类型系统会抛错，是物理防线）

### 3.2 "小红书 ins 风" 翻车版

**画像**：哑光橙 `#ff9966` + 雾粉 `#fcd3c9` + 背景高斯模糊大图 + 每个容器都带 4px 柔和阴影 + 标题字距 0（因为 ins 风爱用挤字标题）+ tip 容器是亮粉色 + warning 是亮黄色。

**规避**：
- **全局无阴影**（box-shadow 除了 pre 代码块的内阴影保留作可读性外，其余容器一律 0 shadow）
- **全局无高斯模糊 / 无渐变**（gradient 一旦出现，ins 风就定性）
- tip / warning 改哑光实物色（菜叶绿、黄铜色），见 §1.1
- 所有标题 **letter-spacing ≥ 1.5px**，h1 达到 4px——ins 风的特征是"字挤"，我们反向操作

### 3.3 "少女心 / 治愈系" 翻车版

**画像**：雾粉 `#fbe4e4` + 薄荷 `#c9e4d6` + 淡紫 `#d6c9e4` + 圆角 12px + 所有图标带笑脸 + danger 容器做成"温柔提示"粉色。

**规避**：
- 整套色板里**完全不含粉色、不含紫色、不含高明度薄荷绿**
- danger 就是 danger，用赤陶深 `#8c4a3f` + 上下粗线——生活美学不负责"温柔化警告"，读者需要严肃信息时必须给严肃信息
- 所有图标**去圆框**——圆框 + 内部符号的组合自带"卡通"气质

### 3.4 "手写体 + 水彩花束" 翻车版

**画像**：h1 仿手写字体（script font）+ divider 全是水彩叶子 PNG + quoteCard 四周画一圈花环 + section-title 前一束彩色小花。

**规避**：
- 禁止任何 background-image / 禁止 PNG 装饰（平台会吞，且手绘风在小屏失真严重）
- 所有"花"都不画，**唯一保留的植物是 dividerWave 里那根几何化带叶枝条**，且整篇最多一次
- quoteCard 四角改为**仅左上 + 右下两枚编辑引号**，中间大留白——而非花环

### 3.5 通用错误：**"生活美学 = 圆角 + 米色"** 的认知偷懒

**这是升格的终极敌人**。规范里必须反复强调：

- **圆角不是生活美学的信号**，质地才是。`radius 0` + 亚麻米 + 陶土褐 + 细铜线 hairline，比任何圆角都更"生活刊物"
- **米色不是万能底色**。注意米底有**冷米（亚麻）** vs **暖米（奶油）** 的根本分野——我们要前者
- **植物不是唯一语汇**。器皿、窗光、布料、纸角、印刷标记都是。砍掉四件植物，换上四件器物，就是这套规范最重要的决定

---

## 结语（给落地工程师）

这份规范的核心只有一句话：**去甜、去圆、去植物、去手绘感；留质地、留几何、留器物、留留白。**

凡是让你想加一个圆角、加一片叶子、加一笔水彩、加一点奶油色的时候，先问：**它是 Kinfolk 会用的吗？它是 Apartamento 的静物台上会出现的吗？** 答不上来，就删。

落地时六件事最重要：

1. 主色改 `#9c6b4f`（陶土褐），secondary 改 `#6b7a72`（青瓷灰绿）——**冷色反转**是色板升格的关键一步
2. bg 改 `#efece4`（亚麻米），所有 radius 归零（sm=0, md=0, lg=2）
3. 删除 dividerFlower（缠枝）、替换 dividerDots（花瓣→小方点或 null）、替换 h2Prefix（叶子→薄切线）、替换 sectionCorner（叶片→信封折角）、替换 quoteMark（豆芽→编辑引号 〔〕）、替换 stepBadge（双环→日课方格）
4. 新增一件植物 motif 到 dividerWave，但必须是**几何化带叶枝条**（菱形叶、无填充、每篇限用一次）
5. admonition 四件套走**四种不同 variant**（minimal-underline / accent-bar / pill-tag / ticket-notch），且四个角色名改为**小记 / 案头 / 留心 / 慎**
6. Typography 走"杂志页眉"路：h1 28/500/4px 字距、p 15/400/1.75 行高、**禁用 700 字重**
