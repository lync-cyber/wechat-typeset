# Literary Humanism · 视觉人格规范

> 本文档是 **设计规范**，不是实现。落地由工程 agent 按此规范生成 tokens / elementOverrides / containers / assets。
> 所有判断都在公众号硬约束下成立：禁 `font-family` / `position` / `float` / flex / grid / `@media` / `@keyframes` / `:hover` / `-webkit-*` / flex gap，布局只有 `table` / `inline-block` / `block` + padding + margin + border。SVG 禁 id / style / script / url()，stroke-width ≥ 1，`<text>` 不声 font-family，光栅后字号 ≥ 14，纯白用 `#fefefe`。

---

## 定位一句话

**人文札记不是"新中式"，是"出版物"。** 参照坐标是三联书店的毛边本、中华书局的竖排繁体集、《读库》正文页、岩波文库的奶白封皮——不是茶馆菜单，不是文旅折扇，不是梅兰竹菊四条屏。气质关键词：**克制、藏锋、留白、纸感**。所有决策都从这条线出发，凡是让版面"热闹"的东西一律先枪毙再说。

## 1. Persona 三件套

### 1.1 Color story

当前配色（驼棕 `#7a5c3a` + 松绿 `#3c4e3a` + 朱砂 `#a94633` + 米底 `#f5f1e8`）的问题不是色相错了，是**饱和度过高、三色等权**——三种强色都在抢正文的注意力，结果谁也不像主角。出版物的颜色纪律是：**一个主色负责压住整页的调性，一个点缀色克制到每篇只出场两三次，其余靠层次灰完成叙事**。

#### 新色值提案

| token | 旧值 | 新值 | 命名（内部沟通用） | 理由 |
|---|---|---|---|---|
| `bg` | `#f5f1e8` | `#f4efe3` | 宣纸米 | 降一点黄味，偏近"未漂白道林纸"。`#f5f1e8` 在屏幕上偏奶油，容易显廉价。 |
| `bgSoft` | `#ede6d4` | `#ece5d1` | 旧纸阴影 | 与主底拉开 8% 明度，足够分辨容器边界又不至于"贴纸条" |
| `bgMuted` | `#e3d9bf` | `#ddd3bb` | 装帧衬布 | 仅用于 code / 边栏深底，不上大面积 |
| `text` | `#2b261c` | `#1f1b14` | 松烟墨 | 更深、更冷、更接近油墨印刷。文字必须比所有装饰色都深，这是出版物 vs 文创的第一条分水岭 |
| `textMuted` | `#6a604c` | `#6b5f4a` | 墨灰 | 微调，保留 |
| `primary` | `#7a5c3a`（驼棕） | `#5a4a3a` | 宋椠褐 | **关键改动**：把"驼棕"往"油墨褐"压。`#7a5c3a` 是皮具色、包装纸色；`#5a4a3a` 是宋版书印刷在黄纸上的墨迹色。拒绝茶馆，拥抱书斋。 |
| `secondary` | `#3c4e3a`（松绿） | `#3d4a3d` | 青瓷釉 | 降 5% 饱和度，绿里掺一点灰。松绿原色太"端午粽叶"，不够"古籍扉页的绢布" |
| `accent` | `#a94633`（朱砂） | `#9a3b2e` | 藏经朱 | 收窄饱和度；这颗朱要**稀有**，每屏最多出现两次（钤印 + 首字 / 钤印 + 批注号） |
| `border` | `#d7ccb2` | `#cfc3a8` | 版框线 | 维持，微调 |
| `code` | `#a94633` | `#6b5f4a`（= textMuted） | 墨灰 | **拒绝让朱砂承担代码色**。代码在人文语境里是"注音符号"，不是"警示"；用墨灰 + bgMuted 底，安静处理 |

#### "拒绝什么"

- **拒绝 `#c8a15a` / `#d4a95c`**（亮金）。这是月饼盒配色，不是书封。
- **拒绝纯正朱红 `#dc143c` / `#e60012`**。那是春联红和日本国旗红，不是藏经朱。
- **拒绝 RGB 互补对冲**（红 + 绿同强度）。当前 primary 和 accent 的彩度差不够大，新配色把 secondary 往灰里压，让 accent 在页面上真正"跳"出来。

#### 语义色（tip / warning / info / danger）

必须满足：米底 `#f4efe3` 上正文 `≥ 4.5:1` 对比度（AA），accent 色本身作为**标题/图标/边框**用，不作为正文。

| 语义 | accent（标题与图标色） | soft（容器浅底） | 对比度（accent on bg） | 角色名 |
|---|---|---|---|---|
| tip | `#4a6b3e` | `#e3e8d6` | 5.1:1 | **批注**（旁批·眉批） |
| warning | `#8a6428` | `#efe3c9` | 4.7:1 | **商榷**（存疑·待考） |
| info | `#3d5a75` | `#dce3ec` | 5.4:1 | **案语**（按：编者补注） |
| danger | `#8e3a2d` | `#ecd6cf` | 5.3:1 | **校勘**（异文·误植） |

#### 松绿 tip 与朱砂 danger 同页不打架——三条纪律

1. **色相距离够远**：tip 偏灰青绿（`#4a6b3e`），danger 偏暗朱红（`#8e3a2d`），彼此离开色轮 120° 以上。而不是当前那种"红配绿都很鲜艳"的民间画法。
2. **强度不对等**：tip 只出现**细线 + 浅底 + 图标**，不用粗色条；danger 才可以用**断口 + 双边框**这类更重的结构。视觉分量差一档，不会互抢。
3. **在同一篇里 danger 的 soft 底**要比 tip 的 soft 底**更深**（`#ecd6cf` vs `#e3e8d6`），让 danger 自动占据"更沉"的版位，读者视线顺序先 tip 后 danger，而非并置。

### 1.2 Typographic voice

不能写 font-family，全部的节奏靠 **size / weight / letter-spacing / color / margin** 拼骨架。这反而是福：出版物的节奏本来就**不靠字体**而靠**字距和行距**。

#### Scale

| 层级 | px | font-weight | letter-spacing | line-height | 颜色 | 说明 |
|---|---|---|---|---|---|---|
| h1（标题） | 26 | 700 | 3px | 1.5 | `text` | 在 h2 之上单独拔高 1px，加大字距到 3px——模仿古籍大字题签，**稀**比**粗**更贵气 |
| h2（章节） | 19 | 600 | 2px | 1.6 | `text` | **比当前降 1px**。h2 原来的 20px 过于端正像教科书；19 + 600 + 2px 字距 = 克制 |
| h3（小节） | 16 | 600 | 1.2px | 1.7 | `text` | **与正文同号**。只靠字重 + 字距区分——这是出版物的高级手法：h3 不吼，读者靠字距识别 |
| p（正文） | 16 | 400 | 1px | 2.0 | `text` | 保留当前的 2.0 行高——这是全主题唯一敢给 2.0 的，是人文札记的签名 |
| small（附注） | 13 | 400 | 0.5px | 1.7 | `textMuted` | 批注 / 日期 / 引文来源 |
| pull-quote（金句） | 22 | 500 | 1.5px | 1.8 | `primary` | **金句必须突兀地放大**。22px 的 500 字重，在 16/400 的正文里会像一块压在纸上的镇纸——这就是我要的效果 |

#### 字重对比策略

禁止用"三级字重"（300 / 400 / 700）堆叠——公众号会吃掉细字重。只用 **400 / 500 / 600 / 700 四档**，且：
- 正文 400，强调 500（不是 700！500 已经够有分量，700 太硬）
- 标题 600 或 700
- **不允许任何 700 + 大字号 + 主色饱和度** 的三元组合出现——那是广告牌，不是书

#### 首字下沉 / 段首缩进

- `quote` 的 `magazine-dropcap` variant 是人文主题的招牌动作：**首字 42px / 700 / `accent` 色 / float 不能用改为 `display:inline-block; margin-right:6px; line-height:1` 配合两行正文**。首字是全篇唯一允许用 accent 色的字符（与钤印共享这颗朱）。
- 正文段落 **不做** `text-indent: 2em`。公众号竖排阅读习惯早已西化，2em 首行缩进反而让版面碎。**靠段间距 22px 的呼吸完成段落分割**。
- `intro`（引子）可以首段 `text-indent: 0` 但**整段左右各加 20px 内缩**，模拟古籍"题解"的内缩行款。

### 1.3 SVG motif

现有资产（如意云头 / 缠枝花 / 回纹 / 书角 / 钤印）**语汇正确但太满**。升级原则一句话：**从"装饰"走向"版心规制"**——留白的重量 > 花饰的重量。

下面是 5 个标志性符号的最终方案：

#### ① 版框双线（取代当前 h2 的"如意云头 + 横线"）

如意云头在 h2 前缀位置出现太频繁会变成"每章节都戴同一顶帽子"。换成：

```
   ┃
   ┃   h2 标题文字
   ┃
   └─────────────────────────
         （h2 下方细版框线）
```

- h2 **前缀改为一根 2px 宽 × 20px 高的 `primary` 色竖条**（SVG 或纯 border-left），替代云头 icon
- h2 **下方**保留 1px `border` 横线，但**不是通栏**——只画到文字宽度的 60%，右侧留白
- **为什么**：竖条 + 截短横线，是《读库》目录的典型做法。去掉云头不是丢弃语汇，而是把云头留给更稀有的位置（dividerFlower）。

SVG 伪代码（文字描述）：`viewBox 4x22`，左侧 `rect x=0 y=0 w=2 h=20 fill=primary`，仅此而已。

#### ② 如意云头·单颗（`dividerFlower` / `sectionCorner` 专用）

把"云头"这个最有辨识度的符号**降频使用**：仅在 `dividerFlower` 和 `sectionCorner` 出现。单颗而非一排。

```
           ╭─╮╭─╮╭─╮
   ────────┘  └┘  └┘  ────────
               ·
              (朱点)
```

- viewBox 降到 `0 0 240 18`（当前 22，收紧）
- 中间仅放**一组三连云头**（当前是五连，太满），宽度 32px，两侧长线各 100px
- 云头下方的朱点 `r=1.5` 而非 `r=1.8`，不喧宾夺主
- stroke-width 1 → 0.9（允许，因为光栅后仍 ≈1px）
- **去匠气**：删除当前版本"下方水平线 + 顶部朱圆 + 连接短线"的三件套，只保留云 + 朱点

#### ③ 版心回纹·单枚（`dividerDots` 取代当前"五个回纹方印"）

当前 dividerDots 把五个回纹方印一字排开，像"印章店招牌"。改为**居中单枚 + 两侧长线**，回纹本身绘制更细：

```
    ──────────────  ▣  ──────────────
                  (单枚回纹方印)
```

- viewBox `0 0 240 12`
- 中央单枚回纹：`rect 10x10` 外框 + 内部 L 形双折线，stroke 1，opacity 0.7
- 两侧 `border` 色细线各 108px，单线（不要当前的双线 + 半透明第二线，过度）
- **理由**：五连回纹是版画扉页用法，过于地域化；单枚回纹是宋版书"鱼尾"位置的用法，更高级

#### ④ 钤印（stepBadge + quoteMark 共享）

当前 stepBadge 用"双圈 + 朱数字"已经非常对，**保留方向**。微调：

- 外圈 `r=11` stroke 1.2（保留）
- 内圈 `r=8.5` stroke 0.6 opacity 0.55（保留）
- 底色浅填充从 `primary 0.12` 降到 `primary 0.08`——更像宣纸透印而非色块
- 数字色由 `accent`（朱）改为 **`primary`**（墨褐）——理由：**朱砂是稀缺资源**，全文留给首字和一枚真正的结尾钤印，不能在 steps 里七八颗朱圆点泛滥

单独新增一个"大钤印"资产 `sealMark`（仅用于 `footerCTA` 末尾）：24×24 方形外框 + 内部两字留位，**这颗才用朱**，作为全文收束。

#### ⑤ 批注号（新增，给 tip / danger / 校勘系用）

仿古籍"夹注"的双鱼尾符号，作为 admonition 容器标题前的统一前缀：

```
   〖  ⟩ 批注
   〖  ⟩ 校勘
```

- 14×14 SVG，`〖` 或 `⟩` 的几何化——两条 2px 弧线开口朝右
- 颜色跟随 status.accent（tip 绿 / danger 朱红 / warning 黄褐 / info 青）
- **作用**：四种 admonition 都用同一形状、**只换颜色 + 换开口方向**，形成系统感

## 2. 19 个 container 的视觉差异化方案

### intro · 卷首语

**角色**：题解。全文的第一次呼吸，告诉读者"接下来要谈什么、从什么角度谈"。

**结构**：
```
        ┃ 引  题解文字，两侧内缩 20px，
        ┃     italic 风格（但禁 font-style:italic
        ┃     以防公众号回退——改用 textMuted + letter-spacing 1.5px
        ┃     造出"倾斜感"）
```

- 左侧 2px `primary` 竖条（与 h2 前缀呼应）
- padding `14px 20px 14px 22px`，margin `0 0 28px 0`
- 底色 `bgSoft`
- 用 svg 资产 `h2Prefix`（那根竖条）在起始位置——不额外造新资产

### author · 作者条

**角色**：署名。短、静、冷。

**结构**：单行 `display:inline-block` 胶囊：`padding 6px 14px`，`bgSoft` 底，`primary` 色小号 14px 文字，**无边框无圆角**（`radius 0`）。
- 人文主题的署名不是"胶囊徽章"，是**书末小字落款**——不要圆角。

### cover · 封面

**角色**：扉页。图片 + 一行题记。

**结构**：
- 图片 `border-radius 0`（出版物封面不圆角）
- 图下紧跟一行 `textMuted / 13px / letter-spacing 2px / text-align:center` 的小字题记
- margin `0 0 36px 0`

### section-title · 章节标题

**角色**：卷/章的分隔，比 h2 更重。

**variant 选择**：`cornered`。
**结构**：
```
   ┌─┐
   │ │  章节名（19px / 600 / 2px 字距）
   └───────────────────────────────
```
- 左上角用 **新版 `sectionCorner`**（书角折页，保留）
- 下方 1px `border` 全通栏
- margin `40px 0 18px 0`（section 级别必须有更大的上 margin）

### quote · 引文（blockquote）

**角色**：征引。别人的话，不是我的话。

**variant 选择**：`column-rule`（双侧细竖线夹住）。这是全主题最克制的引用样式——古籍夹注常用"双行小字"，数字化等价物就是双竖线。
**结构**：
- 左右各一根 1px `primary` 竖线
- padding `8px 22px`
- `textMuted` 色 + letter-spacing 1.2px
- **不加引号 SVG**——引号交给 quoteCard 专属

### quoteCard · 题辞（金句卡）

**角色**：题辞。**这是全篇最贵的一页**，用朱、用大字、用缩进。

**variant 选择**：`magazine-dropcap`。
**结构**：
```
   ╭────────────────────────────╮
   │                            │
   │  大  首字 42px accent 色    │
   │  题辞正文 22px primary 500  │
   │  letter-spacing 1.5px       │
   │                            │
   │                    ——出处   │
   ╰────────────────────────────╯
```
- 外框 1px `border`，`radius 0`（**不圆角**）
- padding `28px 24px`
- 底色 `bgSoft`
- 首字下沉 42px / 700 / accent
- 出处小字 13px / textMuted / 右对齐 / 前缀 `——`
- **不用** `quoteMark` SVG——首字本身已经是装饰，再加引号就太满

### footerCTA · 卷尾

**角色**：终。收束用，不是"关注我"。

**结构**：
- 顶部 `dividerFlower`（单云头）
- 中间文字 13px textMuted 居中
- 底部右下角放一枚新增的 `sealMark`（大钤印 24×24，朱色）
- padding `28px 0 20px 0`，无底色、无边框

### recommend · 延伸阅读

**角色**：卷末书单。

**结构**：`bgSoft` 底框，每条一行，行前加一个 `▸` 字符或 3×6 三角 SVG（`border` 色）。
- padding `16px 20px`，radius 0
- 标题行 `primary / 600 / 14px`："延伸阅读"
- 列表项 14px / line-height 1.9

### qrcode · 二维码

**角色**：钤印衬托。

**结构**：居中 block，图像下方一行 13px textMuted 说明（"扫码订阅"之类）。图像 `border 1px border-color`，`padding 8px` 白色内衬（**用 `#fefefe`，非 `#fff`**）。

---

### tip / warning / info / danger · 四件套（**重头戏**）

**核心原则：不能只靠色差区分**。四者在形状策略、边框样式、图标开口方向上全都不同。即便打印成灰度，读者也能一眼分出哪个是哪个。

| 语义 | variant | 形状策略 | 边框 | 标题前缀 | 留白 |
|---|---|---|---|---|---|
| tip（批注） | `minimal-underline` | **最轻**。无边框，仅标题下方 1px `tip.accent` 色短线（标题宽度的 80%） | 无 | `〖批注〗` + `〗` 开口向右图标 | padding `4px 0 12px 0`，**无底色** |
| info（案语） | `accent-bar` | 左侧 3px `info.accent` 竖条 + `info.soft` 浅底 | 无外框 | `按：` 两字加粗 + 无图标（低调） | padding `12px 16px 12px 20px` |
| warning（商榷） | `pill-tag` | 顶部贴一枚"存疑"胶囊（悬于顶边上沿，靠 margin-top 负值） + 外框 1px `warning.accent` | 1px 虚线风格用 dashed 不稳定改为 1px + `warning.soft` 底 | 胶囊内"商榷"二字，13px 700 `textInverse` | padding `16px 18px` |
| danger（校勘） | `ticket-notch` | **最重**。左右票根缺口 + 双边框（外 1px `danger.accent` 实线 + 内 1px `bgSoft` 留白 + 再内 0.5px `danger.accent` 虚化） | 双线 | `校』` + `』` 开口向左图标 | padding `18px 24px` |

**一眼可辨的锚点**：
- tip 是**最轻**的（无边框，像书页上一道铅笔划）
- info 是**最窄**的（左竖条，像旁批）
- warning 是**最跳**的（顶部胶囊，悬出边界）
- danger 是**最重**的（双框 + 缺口，像勘误贴条）

四者的标题前缀字符（批注 / 按 / 商榷 / 校勘）也**从语言上**强化差异——这比色块更有效。

---

### highlight · 高亮段

**角色**：着重段。比 p 醒目，比 quoteCard 克制。

**结构**：`bgSoft` 底 + 左右各 24px 内缩 + 无边框 + letter-spacing 1.5px。整段文字色仍是 `text`，但**字重升到 500**。不用图标、不用边框——**靠底色 + 内缩 + 加粗三件事完成重音**。

margin `24px 0`，padding `14px 24px`。

---

### compare · 夹注（人文主题不做"左右对阵"）

**角色**：**夹注 / 对勘**，不是 PK。两家说法并置，让读者自判。

**variant 选择**：`column-card`，但语义改造。
**结构**：
```
   ┌──── 甲说 ────┐   ┌──── 乙说 ────┐
   │              │   │              │
   │   论述内容    │   │   论述内容    │
   │              │   │              │
   └──────────────┘   └──────────────┘
```
- 两栏 `display:table-cell`，中间 16px 空隙（用 margin，不用 gap）
- 每栏 `border 1px border-color`，`radius 0`，padding `16px 18px`
- 标题行用 `primary / 14px / 500`，**前缀文字改为"甲说 / 乙说"或"正说 / 反说"**（而不是"优点 / 缺点"这类现代文案）
- **拒绝 `ledger` variant**（绿红双色账本）——那是效率软件审美，不是人文

---

### steps · 卷次（不是 1→2→3）

**角色**：**卷一 / 卷二 / 卷三**——时间或层次的递进，不是流程图。

**variant 选择**：`timeline-dot`。
**结构**：
```
   ●  卷一 · 标题
   │   正文内容，左侧 24px 留白让
   │   正文与圆点对齐
   │
   ●  卷二 · 标题
   │   ...
   │
   ●  卷三 · 标题
```
- 左侧 20px 宽的列放 stepBadge（钤印，数字色改为 `primary` 墨褐）
- 圆点之间用 1px `border` 色竖线连接（普通 border-left 实现）
- 每步标题前缀**由渲染器加"卷一 / 卷二 / 卷三"**（走 template 配置，用户可覆盖）
- **禁止** `ribbon-chain` variant（横向色带链）——那是课程卡片审美

---

### divider · 分割

**角色**：呼吸。比段落间距更大一级的停顿。

**variant 推荐**：默认 `flower`（单云头），次选 `dots`（单枚回纹），`wave`（缠枝）留给章节级别，`rule` 和 `glyph` 不推荐。

**为什么**：
- `flower` 最符合 literary-humanism 的签名气质——云头是这个主题的 logo
- `dots` 备选是因为有些文章节奏密集，需要更短的停顿
- `wave`（缠枝）原来太满，升级后当作 **section 级别**的超大分隔（每 3-4 个 h2 之间用一次，不是每个段之间）
- `rule` 是默认主题的做法，放在人文主题里偷懒气太重，否决

所有 divider 的上下 margin **从当前的默认值提到 `36px 0`**——人文主题需要更大的呼吸。

---

### mpvideo / mpvoice · 微信原生组件

**角色**：嵌入件。这两个是平台强制样式，主题能做的只有**外围容器**。

**结构**：包一层 `margin 24px 0`，上下各一根 1px `border` 色横线（通栏），模拟古籍"图版"的版框。**不加底色、不加圆角、不加 shadow**——让平台组件本身的外观保持干净。

### free · 自由容器

**角色**：兜底。

**结构**：padding `16px 20px`，margin `20px 0`，**无边框无底色无图标**。使用者自己决定内容，主题不预设视觉意见。

---

## 3. 反例对照（如果做糟会长什么样）

一个做糟的 literary-humanism 会是这样——我要在规范里把它钉在耻辱柱上，让任何落地决策都能回头对照：

### 糟糕版本画像

**"古风茶楼套餐"**——大红大金 + 书法体背景水印 + 梅兰竹菊四条角花 + 每个 h2 都戴一顶云头帽 + danger 容器做成仿古告示牌（"通缉"审美）+ footer 放一个盖满整屏的巨型朱砂印章 + quoteCard 边框用五种花纹嵌套 + 背景贴一张半透明山水画 PNG + 主色直接上 `#c8102e` 正红 + 金色 `#d4a017` 点缀一切边框。

### 具体犯的错 & 我的规避

1. **"符号通货膨胀"**——每个元素都带 SVG 装饰，结果没有一个装饰是"重要"的。
   **规避**：朱砂色全文只出现**两处**（quoteCard 首字 + footer 钤印），云头 SVG 全文最多出现**三次**（一次大 divider + 两次小 sectionCorner），stepBadge 改用墨褐不用朱。**稀缺制造贵气，泛滥只剩廉价。**

2. **"书法背景图"**——以为加一张毛笔字 PNG 水印就"文化"。
   **规避**：禁止 background-image。人文气质靠**字距 + 行距 + 留白 + 克制用色**造出，不靠贴图。平台会把 background-image 丢掉，靠这个撑气质的主题一上手机就崩。

3. **"红配绿同饱和度"**——朱砂 accent 和松绿 secondary 当前彩度相当，会在同页互撞。
   **规避**：松绿压灰（`#3d4a3d`），朱砂收窄（`#9a3b2e`）且限制出场次数。彩度拉开差距，让朱真正"跳"。

4. **"h1 居中 + 大字号 + 主色 + 全体粗体"**——公众号文章最俗的开头。
   **规避**：h1 用 `text` 色不用 `primary`，字重 700 但字号只 26px（不是 32+），letter-spacing 3px 靠**稀**而非**粗**。

5. **"圆角 8px 当可爱"**——所有容器都 radius 8 是 SaaS 卡片审美。
   **规避**：所有容器默认 `radius 0`。圆角仅在 intro/author 极窄范围出现，且不超过 2px。**方版心是书，圆角是卡片。**

6. **"admonition 四色同形只换颜色"**——这是默认主题的偷懒做法。在人文主题里不可原谅。
   **规避**：tip / info / warning / danger **四种形状策略**（minimal-underline / accent-bar / pill-tag / ticket-notch）+ 四种标题用语（批注 / 按 / 商榷 / 校勘）+ 四种图标开口方向。打印成黑白也分得出。

7. **"行距 2.0 但字号 15 还挤"**——当前 baseSize 16 + line-height 2.0 是对的，但如果谁把 baseSize 退回 15 省空间，整个人文气质会塌。
   **规避**：`baseSize 16` 是红线，不许动。宁可内容少一屏，也不让行距挤。

8. **"letter-spacing 全局 0.5px 保险做法"**——那是默认主题的审美底线。
   **规避**：正文 1px，标题层级从 1.2 → 2 → 3 px 逐级放大。**字距是人文主题的声带**，不敢给字距的主题没资格叫 literary-humanism。

## 结语（给落地工程师）

这份规范的核心只有一句话：**用"减法"做贵气，用"稀缺"做重点，用"留白"做呼吸。**

凡是让你想加一个装饰、加一条边框、加一颗朱点的时候，先问：**它是稀缺的吗？它替代了什么？** 能答上来再加。答不上来，就留白。

落地时四件事最重要：
1. 把 `primary` 改成 `#5a4a3a`，`accent` 改成 `#9a3b2e`，并在 code path 里删除所有"朱砂用于非钤印位置"的地方
2. admonition 四件套**必须走四种不同 variant**，不能同一个 variant 换颜色
3. stepBadge 的数字色从 `accent` 改为 `primary`
4. 所有容器默认 `border-radius: 0`
